# 高校智慧交流论坛 — 需求分析

## 一、项目背景

当前高校学生的交流场景高度碎片化：课程问题散落在微信群，求职经验藏在私聊记录，跨校交流几乎为零。现有平台（知乎、牛客、微信群）均非专为高校设计，缺乏学生身份体系、跨校协作机制和智能内容服务。

本项目面向全国普通高校在校学生，构建一个集学术交流、校园生活、职业发展、技术问答于一体的智慧论坛平台，核心差异化能力为 **AI 驱动的内容摘要生成与个性化推送机制**。

---

## 二、目标用户

| 用户类型 | 典型需求 |
|---------|---------|
| 大一新生 | 适应校园、了解专业、寻找资源 |
| 在校学生 | 课程互助、校园活动、技术学习 |
| 高年级学生 | 考研经验、竞赛组队、实习求职 |
| 研究生 | 学术讨论、论文方法、导师评价 |

## 三、功能需求

### 3.1 核心功能模块

#### 用户系统
- 注册（邮箱/手机号）、登录、找回密码
- 学校认证（选择学校 + 学生邮箱验证）
- 个人主页（发帖记录、关注/粉丝、积分）
- 用户等级体系（积分驱动）

#### 内容板块（4大类）
| 板块 | 子分类示例 |
|------|-----------|
| 学术交流 | 课程互助、考研备考、论文讨论、竞赛交流 |
| 校园生活 | 活动通知、二手交易、失物招领、美食推荐 |
| 职业发展 | 实习内推、求职经验、竞赛组队、简历互评 |
| 技术问答 | 编程求助、项目展示、工具分享、Bug求助 |

#### 帖子系统
- 发帖（富文本编辑器、图片上传、代码块）
- 回复、点赞、收藏、举报
- 置顶、加精、热帖榜

#### 搜索系统
- 全文搜索（标题+内容+标签）
- 按板块、时间、热度筛选

#### 通知系统
- 站内通知（被回复、被点赞、被关注）
- 未读消息角标

#### 管理后台
- 用户管理、帖子审核、违规处理
- 数据统计（日活、发帖量、热门话题）

---

### 3.2 AI 核心功能（创新点）

#### AI-1：智能摘要生成
- 用户发帖后，调用 DeepSeek API 自动提取50字内摘要
- 摘要展示在列表页，提升浏览效率
- 同时自动生成3-5个标签

#### AI-2：个性化推送机制
- 用户画像：基于浏览历史、点赞、收藏、所在板块建立兴趣标签权重
- 推送算法：内容标签与用户兴趣标签余弦相似度匹配
- 首页「为你推荐」区域每次登录动态刷新

#### AI-3：智能问答助手
- 论坛内嵌 AI 助手浮窗（调用 DeepSeek）
- 用户可直接提问，AI 结合论坛内已有帖子内容回答
- 支持追问上下文

#### AI-4：内容质量检测
- 发帖时 AI 检测是否含违规词、广告内容
- 给出内容质量评分，低分帖子进入人工审核队列

## 四、非功能需求

| 指标 | 目标 |
|------|------|
| 响应时间 | 普通接口 < 500ms，AI接口 < 3s |
| 并发 | 支持50人同时在线（演示阶段） |
| 安全性 | 密码bcrypt加密，接口JWT鉴权，SQL防注入 |
| 兼容性 | Chrome/Edge/Firefox 最新版 |
| 可扩展性 | 模块化API设计，便于后续增加功能 |

## 五、竞品分析

| 维度 | 知乎 | 牛客网 | CC98（浙大） | **本项目** |
|------|------|--------|-------------|-----------|
| 用户范围 | 全社会 | 技术/求职 | 单校区 | **全国高校** |
| 身份认证 | 无 | 无 | 校内账号 | **学校邮箱认证** |
| 内容分类 | 综合 | 技术为主 | 综合 | **四大高校专属板块** |
| AI摘要 | 无 | 无 | 无 | **✓ DeepSeek自动生成** |
| 个性化推送 | 有（算法黑盒） | 无 | 无 | **✓ 可解释兴趣标签推送** |
| AI问答 | 无 | 无 | 无 | **✓ 内嵌AI助手** |
| 跨校交流 | 有但无专属 | 有但无专属 | 无 | **✓ 专属高校社区** |

---

## 六、创新总结

1. **高校专属身份体系**：学校邮箱认证，保证社区纯净度
2. **AI摘要+标签自动化**：降低发帖门槛，提升内容可发现性
3. **可解释的个性化推送**：用户可看到推送原因（"因为你关注了Python"）
4. **AI助手与论坛内容融合**：AI回答时引用站内相关帖子，形成知识闭环
